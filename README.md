# Mastersaf Invoice Download Scripts

This repository contains PowerShell scripts to download XML and PDF invoices from Thomson Reuters Mastersaf using the Mastersaf REST API.

## Scripts and execution order

Run the scripts in this order:

1. `Mastersaf-Webservice-ReceptorCTe-build.ps1`
2. `Mastersaf-Webservice-ReceptorCTe-release.ps1`
3. `Mastersaf-ErrorReport-SendEmail.ps1`

## What each script does

### 1) Build script
**File:** `Mastersaf-Webservice-ReceptorCTe-build.ps1`

Purpose:
- Authenticates in Mastersaf.
- Queries invoice keys for a company and date range.
- Creates TXT artifact files with invoice keys.
- Creates `Invoice_ErrorList.csv` for non-authorized/non-canceled invoice statuses.

Parameters:
- `Company` (valid values in current script: `NOVA`, `ZILO`, `TREX`, `LUMA`)
- `UserName`
- `Userkey`
- `MastersafWebServiceURL`
- `DaysInThePast` (example: `-1`)

Environment variable used:
- `BUILD_ARTIFACTSTAGINGDIRECTORY` (output folder for generated TXT/CSV artifacts)

### 2) Release script
**File:** `Mastersaf-Webservice-ReceptorCTe-release.ps1`

Purpose:
- Reads invoice keys generated by the build stage.
- Downloads XML or PDF invoice documents from Mastersaf REST API.
- Writes downloaded files to configured destinations.
- Updates history files to avoid duplicate downloads.

Parameters:
- `Company`
- `UserName`
- `Userkey`
- `CompanyCNPJ`
- `CompanyIE`
- `SavingFolderName`
- `ApplicationName`
- `MastersafWebServiceURL`
- `XMLSavingFolderPath`
- `PDFSavingFolderPath`
- `LogFolderPath`
- `HistoryFileFolderPath`
- `FileType` (`XML` or `PDF`)
- `TXTBuildDropPath` (path to the build artifact folder that contains the generated TXT file)
- `EnableSapLes` (optional)

Note:
- `TXTBuildDropPath` must be provided by the caller.

### 3) Error report script
**File:** `Mastersaf-ErrorReport-SendEmail.ps1`

Purpose:
- Checks if `Invoice_ErrorList.csv` exists in the release artifact folder.
- Sends e-mail report with CSV attachment.
- Runs e-mail logic only between `00:00` and `08:00` local time.

Note:
- The send e-mail step uses an on-premises SMTP server. The machine or agent running the script must have access to the company SMTP server for delivery to work.

Environment variables used:
- `Body`
- `Subject`
- `FromEmailAddress`
- `ToEmailAddresses`
- `SmtpServer`
- `ReleaseArtifactName`
- `TXTBuildDropPath`

## Running from command line

Open PowerShell in the repository folder and run:

```powershell
# 1) Build stage
$env:BUILD_ARTIFACTSTAGINGDIRECTORY = "C:\temp\mastersaf\build-artifacts"
.\Mastersaf-Webservice-ReceptorCTe-build.ps1 `
  -Company "NOVA" `
  -UserName "your-user" `
  -Userkey "your-key" `
  -MastersafWebServiceURL "https://your-mastersaf-host/webservice" `
  -DaysInThePast "-1"

# 2) Release stage (XML)
.\Mastersaf-Webservice-ReceptorCTe-release.ps1 `
  -Company "NOVA" `
  -UserName "your-user" `
  -Userkey "your-key" `
  -CompanyCNPJ "78150693422047" `
  -CompanyIE "6248091735" `
  -SavingFolderName "NOVA" `
  -ApplicationName "Mastersaf-Webservice-ReceptorCTe-Daily" `
  -MastersafWebServiceURL "https://your-mastersaf-host/webservice" `
  -XMLSavingFolderPath "C:\temp\mastersaf\xml" `
  -PDFSavingFolderPath "C:\temp\mastersaf\pdf" `
  -LogFolderPath "C:\temp\mastersaf\logs" `
  -HistoryFileFolderPath "C:\temp\mastersaf\history" `
  -TXTBuildDropPath "C:\temp\mastersaf\build-artifacts" `
  -FileType "XML"

# 2b) Release stage (PDF)
.\Mastersaf-Webservice-ReceptorCTe-release.ps1 `
  -Company "NOVA" `
  -UserName "your-user" `
  -Userkey "your-key" `
  -CompanyCNPJ "78150693422047" `
  -CompanyIE "6248091735" `
  -SavingFolderName "NOVA" `
  -ApplicationName "Mastersaf-Webservice-ReceptorCTe-Daily" `
  -MastersafWebServiceURL "https://your-mastersaf-host/webservice" `
  -XMLSavingFolderPath "C:\temp\mastersaf\xml" `
  -PDFSavingFolderPath "C:\temp\mastersaf\pdf" `
  -LogFolderPath "C:\temp\mastersaf\logs" `
  -HistoryFileFolderPath "C:\temp\mastersaf\history" `
  -TXTBuildDropPath "C:\temp\mastersaf\build-artifacts" `
  -FileType "PDF"

# 3) Error report stage
.\Mastersaf-ErrorReport-SendEmail.ps1 `
  -Body "Invoice processing report" `
  -Subject "Mastersaf Invoice Error Report" `
  -FromEmailAddress "no-reply@company.com" `
  -ToEmailAddresses "ops@company.com" `
  -SmtpServer "smtp.company.com" `
  -ReleaseArtifactName "drop" `
  -TXTBuildDropPath "C:\temp\mastersaf\drop"
```

## Running in Azure DevOps pipeline

Example YAML with build -> release -> errorReport order:

```yaml
trigger: none

pool:
  vmImage: 'windows-latest'

variables:
  Company: 'NOVA'
  MastersafWebServiceURL: 'https://your-mastersaf-host/webservice'
  DaysInThePast: '-1'
  ApplicationName: 'Mastersaf-Webservice-ReceptorCTe-Daily'
  XMLSavingFolderPath: '$(Pipeline.Workspace)\\xml'
  PDFSavingFolderPath: '$(Pipeline.Workspace)\\pdf'
  LogFolderPath: '$(Pipeline.Workspace)\\logs'
  HistoryFileFolderPath: '$(Pipeline.Workspace)\\history'
  TXTBuildDropPath: '$(Pipeline.Workspace)\\drop'
  CompanyCNPJ: '78150693422047'
  CompanyIE: '6248091735'
  SavingFolderName: 'NOVA'

stages:
- stage: Build
  jobs:
  - job: BuildInvoiceKeys
    steps:
    - task: PowerShell@2
      displayName: Build invoice key artifacts
      inputs:
        targetType: filePath
        filePath: Mastersaf-Webservice-ReceptorCTe-build.ps1
        arguments: >-
          -Company "$(Company)"
          -UserName "$(UserName)"
          -Userkey "$(Userkey)"
          -MastersafWebServiceURL "$(MastersafWebServiceURL)"
          -DaysInThePast "$(DaysInThePast)"
      env:
        BUILD_ARTIFACTSTAGINGDIRECTORY: $(Build.ArtifactStagingDirectory)

    - publish: $(Build.ArtifactStagingDirectory)
      artifact: drop

- stage: Release
  dependsOn: Build
  jobs:
  - job: DownloadInvoices
    steps:
    - download: current
      artifact: drop

    - task: PowerShell@2
      displayName: Download XML invoices
      inputs:
        targetType: filePath
        filePath: Mastersaf-Webservice-ReceptorCTe-release.ps1
        arguments: >-
          -Company "$(Company)"
          -UserName "$(UserName)"
          -Userkey "$(Userkey)"
          -CompanyCNPJ "$(CompanyCNPJ)"
          -CompanyIE "$(CompanyIE)"
          -SavingFolderName "$(SavingFolderName)"
          -ApplicationName "$(ApplicationName)"
          -MastersafWebServiceURL "$(MastersafWebServiceURL)"
          -XMLSavingFolderPath "$(XMLSavingFolderPath)"
          -PDFSavingFolderPath "$(PDFSavingFolderPath)"
          -LogFolderPath "$(LogFolderPath)"
          -HistoryFileFolderPath "$(HistoryFileFolderPath)"
          -TXTBuildDropPath "$(TXTBuildDropPath)"
          -FileType "XML"

    - task: PowerShell@2
      displayName: Download PDF invoices
      inputs:
        targetType: filePath
        filePath: Mastersaf-Webservice-ReceptorCTe-release.ps1
        arguments: >-
          -Company "$(Company)"
          -UserName "$(UserName)"
          -Userkey "$(Userkey)"
          -CompanyCNPJ "$(CompanyCNPJ)"
          -CompanyIE "$(CompanyIE)"
          -SavingFolderName "$(SavingFolderName)"
          -ApplicationName "$(ApplicationName)"
          -MastersafWebServiceURL "$(MastersafWebServiceURL)"
          -XMLSavingFolderPath "$(XMLSavingFolderPath)"
          -PDFSavingFolderPath "$(PDFSavingFolderPath)"
          -LogFolderPath "$(LogFolderPath)"
          -HistoryFileFolderPath "$(HistoryFileFolderPath)"
          -TXTBuildDropPath "$(TXTBuildDropPath)"
          -FileType "PDF"

- stage: ErrorReport
  dependsOn: Release
  jobs:
  - job: SendErrorEmail
    steps:
    - task: PowerShell@2
      displayName: Send invoice error report e-mail
      inputs:
        targetType: filePath
        filePath: Mastersaf-ErrorReport-SendEmail.ps1
        arguments: >-
          -Body "$(EmailBody)"
          -Subject "$(EmailSubject)"
          -FromEmailAddress "$(FromEmailAddress)"
          -ToEmailAddresses "$(ToEmailAddresses)"
          -SmtpServer "$(SmtpServer)"
          -ReleaseArtifactName "drop"
          -TXTBuildDropPath "$(Pipeline.Workspace)\\drop"
```

## Notes

- Keep `UserName` and `Userkey` in secure pipeline variables (secret variables or variable groups).
- Pass sanitized paths and company values through pipeline variables or command-line arguments.
